# Output Modes

只有当用户明确要求固定格式时，才使用这些模式。

## Available Modes

### 1. `meeting-notes`
适合：用户明确要“会议纪要 / 圆桌纪要 / roundtable notes”。

建议包含：
- 议题
- 专家面板
- 主要分歧
- 关键判断
- 结论 / 后续问题

### 2. `design-review`
适合：产品、平台、系统、架构类问题。

建议包含：
- 核心判断
- 模块拆分
- 风险与边界
- MVP / 路线建议

### 3. `comparison-brief`
适合：对比型问题。

建议包含：
- 比较维度
- 各专家最强观点
- 共识 / 分歧
- 综合判断

### 4. `standard-template`
适合：用户明确说“标准模板 / 非常标准 / 固定模版”。

只有在这种情况下，才使用更强结构化的格式。

## Rule

如果用户没有明确要求模板，就不要机械套模板。
