# Expert Selection

核心任务不是“凑几个会说话的人”，而是找到真正能把问题讲深的人。

## Prioritize in This Order

1. 直接建设者
   - 做出关键产品、协议、系统或标准的人。

2. 长期研究者
   - 长期围绕该议题写作、研究、建模、解释的人。

3. 风险与反对视角
   - 专门揭示失败模式、攻击面、制度代价、治理问题的人。

4. 落地运营者
   - 把理念变成真实产品、分发、组织、营收或基础设施的人。

5. 外部但相关的意外视角
   - 不在核心圈层里，但能指出更大系统约束的人。

## Good Panel Properties

一个好的 panel 通常具备：

- `直接相关`：每位专家都和问题核心直接相连；
- `有张力`：不是同一派换皮；
- `有层次`：既有理念，也有执行，也有风险；
- `可综合`：分歧足够明显，但不是纯噪声。

## Avoid by Default

- 泛岗位 persona；
- 固定复用某个领域的“明星 panel”；
- 纯名气导向但与问题弱相关的人；
- 观点高度重合的 panel；
- 只选“支持者”，不选能指出限制的人。

## Fallback Order

如果真实个人专家不够，按以下顺序降级：

1. 机构 / 团队
2. 学派 / 路线
3. 方法论视角

降级时要明说，不要假装还是“专家本人”。

## Expert Card

每位专家建议至少压缩成 6 个字段：

- 姓名 / 机构 / 学派
- 身份
- 与议题相关的公开工作
- 核心立场
- 看问题的镜头
- 最擅长判断什么
- 为什么必须入选

如果空间允许，再补一个字段：

- 可能忽略什么 / 可能带来的偏置

## Minimum Introduction Rule

无论最终输出是不是模板，都不要只报人名。

至少要让用户知道：

1. 这个人是谁；
2. 为什么和当前议题直接相关；
3. 他会优先从什么角度看这个问题。

## Topic-First Rule

每次都从当前议题重新建 panel。

不要因为上一题是 web3、AI、哲学、经济学或安全，就把那套专家名单沿用到下一题。
